# PDF Translate — Implementation Plan

## 1. Project Overview

A desktop application that uses LLMs (initially DeepSeek) to translate academic papers from English into Chinese, preserving the original PDF layout. The UI is inspired by Zotero's PDF reader. Core interactive features include:

- **Sentence selection → highlight**: selecting a sentence in the original text highlights the corresponding translated segment in the translation panel.
- **Word selection → pop-up**: selecting a single word opens a pop-up showing its definition and relevant conceptual knowledge (retrieved or generated by the LLM).
- **API key configuration**: users enter their own LLM API key; DeepSeek is the first supported provider.

---

## 2. Tech Stack

| Layer | Choice | Rationale |
|---|---|---|
| Runtime | Python 3.11+ | Rich PDF/NLP ecosystem |
| GUI framework | PyQt6 (or PySide6) | Native desktop feel, good PDF widget support |
| PDF rendering | `pypdf` + `pdfplumber` (parsing) · `pymupdf (fitz)` (rendering & annotation) | MuPDF gives pixel-accurate rendering close to Zotero |
| LLM client | `openai` SDK (DeepSeek is OpenAI-compatible) | Simple swap of `base_url` to add providers |
| Translation cache | SQLite via `sqlite3` stdlib | Persist translations across sessions without a server |
| Packaging | PyInstaller | Single-executable distribution |

## 4. Core Features — Detailed Design

### 4.1 PDF Rendering & Layout Preservation

- Use **PyMuPDF (`fitz`)** to render each page as a high-DPI pixmap displayed in a `QLabel` or custom `QWidget`.
- Extract text blocks with bounding boxes (`page.get_text("dict")`), building an in-memory index: `{page_num: [{block_id, bbox, text, spans}]}`.
- The translation panel renders the translated text in blocks that mirror the original bounding-box positions, creating a side-by-side or overlay view (user-switchable).

### 4.2 Sentence Selection → Translation Highlight

1. User clicks and drags on the original PDF panel.
2. `SelectionHandler` resolves which text spans fall within the mouse rectangle using the bounding-box index.
3. The selected spans are grouped into a logical sentence (split on `.`, `?`, `!`, or paragraph boundary).
4. The handler emits a `sentence_selected(sentence_id)` signal.
5. The Translation Panel listens and highlights the corresponding translated block (same `sentence_id`).
6. If the translation for that sentence does not yet exist in cache, `TranslationService` is called and the result is streamed into the panel in real-time.

### 4.3 Word Selection → Pop-up Definition

1. When `SelectionHandler` detects a **single token** (no spaces, or double-click), it emits `word_selected(word, context_sentence)`.
2. A `WordPopup` (`QFrame` overlay) appears near the cursor.
3. `WordLookupService` checks the SQLite cache first; on miss, sends a prompt to the LLM:
   ```
   Define the term "{word}" as used in the context:
   "{context_sentence}"
   Give: 1) a concise definition, 2) relevant background concepts in 2-3 sentences. Reply in Chinese.
   ```
4. The response streams into the pop-up. The pop-up closes on click-outside or Escape.

### 4.4 Translation Service

- **Segmentation**: `pdfplumber` extracts paragraphs; a sentence splitter (regex + `spacy` optional) breaks them into sentence units, each assigned a stable `sentence_id = hash(page+block+sentence_index)`.
- **Batching**: sentences are sent to the LLM in paragraph-level batches to preserve context. Each batch prompt:
  ```
  Translate the following academic English text to Chinese.
  Preserve technical terms. Return only the translation, paragraph by paragraph,
  keeping the same number of paragraphs as the input.

  [text]
  ```
- **Streaming**: uses the OpenAI SDK's `stream=True`; tokens are pushed to the UI via Qt signals.
- **Cache**: results stored in SQLite as `(sentence_id TEXT PRIMARY KEY, original TEXT, translation TEXT, model TEXT, timestamp INTEGER)`.

### 4.5 LLM Client Wrapper

```python
class LLMClient:
    PROVIDERS = {
        "deepseek": {"base_url": "https://api.deepseek.com/v1", "default_model": "deepseek-chat"},
        "openai":   {"base_url": "https://api.openai.com/v1",   "default_model": "gpt-4o"},
    }

    def __init__(self, provider: str, api_key: str): ...
    def translate(self, text: str, stream: bool = True) -> Iterator[str]: ...
    def define_word(self, word: str, context: str) -> Iterator[str]: ...
```

### 4.6 Settings & API Key Management

- **Settings Dialog** (`QDialog`): provider dropdown, API key field (masked input), model name override, max tokens, temperature.
- API key stored in the OS keychain via the `keyring` library (never written to plain config files).
- `QSettings` stores non-sensitive preferences (window layout, last file, provider choice).

## 8. Key Technical Challenges & Mitigations

| Challenge | Mitigation |
|---|---|
| PDF text extraction accuracy (columns, footnotes, math) | Use `pdfplumber` for structure + `fitz` for rendering; fall back to OCR via `pytesseract` for scanned PDFs |
| Aligning translated blocks with original layout | Store bbox metadata per sentence; use absolute positioning in the translation panel |
| Sentence boundary detection in academic text (abbreviations, citations) | Regex + lookup table for common abbreviations (`et al.`, `Fig.`, `Eq.`) before splitting |
| LLM rate limits on large papers | Queue with exponential back-off; translate page-by-page on demand, not all at once |
| API key security | `keyring` library writes to OS credential store (Keychain / Windows Credential Manager / SecretService) |
| Streaming UI updates from background thread | Worker `QThread` + `pyqtSignal` to push tokens safely to the main thread |
